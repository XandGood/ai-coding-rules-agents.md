# AI 编码助手协作规则 (AI Coding Rules for Agents.md)

## 说到底只是一段提示词，只能说可以一定程度上优化AI编码过程，难以避免存在问题与疏漏，欢迎指正。

本项目编写了一套标准化的规则集，旨在指导AI助手（如OpenCode）更高效、更安全地参与软件编码任务。这些规则部分直接来自于网络，部分由本人进行了一些修改与添加。

## 🎯 背景与目标

在与AI协作编码时，常常遇到以下痛点：
- AI直接执行危险命令（如删除文件、修改数据库）。
- AI在不明确需求的情况下直接编写代码，导致返工。
- 代码完成后缺乏有效的测试建议。
- 代码提交流程不规范。

本规则集正是为了优化这些问题而设计。但受限于大模型能力与幻觉，提示词还是没办法让AI时刻保持清醒的注意，仍建议在进行关键或危险操作时人盯住！！

## ⚠ 工作规则

### 重要原则

- **全部用中文回答用户，称呼用户为老板**
- 此处可以放置一些其他的项目核心信息

### 基本原则

1. **当需要运行危险或者关键命令时，应当告知老板交由老板确认或操作**，避免不必要的自行操作
2. **在编写任何代码之前，请先描述你的方案并等待批准**，如果需求不明确，务必提出澄清问题
3. **如果一项任务需要删除超过 3 个文件，请先停下来**，将其分解成更小的任务并提醒一下老板
4. **当修改代码后没有产生预期效果需要完全重新修改或者老板选择切换其他方案时，注意恢复原状后再进行改动**
5. **当需要代码示例时可以寻找已完成的同类型文件作为示例，模仿其包括但不限于代码风格、注解顺序等所有内容**
6. **遵循行业规范与最佳实践**，包括官方文档推荐和行业惯用思路
7. **文档存放规则**：在 docs 目录下，所有文档（除了 AGENTS.md）都存放在该目录下
8. **编写代码后，列出可能出现的问题**，并**建议**相应的测试用例来覆盖这些问题
9. **Git提交规范**，当任务需要git提交时，创建独立分支进行提交，合并主分支应由老板完成
10. **每次老板纠正方法上的错误之后，就在 AGENTS.md 文件中添加一条对应的新规则**
---

称呼主要是为了便于观测上下文长度是否足够，可以随意更换。
类似于老板之类的称呼似乎会导致AI不敢质疑你的提示词，发出疑问和需求后直接盲从导致错误，可以考虑加上强制质疑之类的原则。

前六条是相对最推荐的部分，后四条可以根据自己需求选择加入。所有原则都可以自行根据需求进行完善和修改。
